GitHub 저장소 생성 및 Personal Access Token(PAT) 발급 안내
백업 시스템을 업로드하려면 코드를 저장할 **저장소(Repository)**가 필요합니다. 아래 안내에 따라 새 저장소를 만들고, 접근에 필요한 **Personal Access Token(PAT)**을 발급받아 저에게 알려주세요.
1. 새 저장소(Repository) 만들기
첨부해 주신 이미지의 GitHub 화면 좌측 상단에 있는 녹색 'New' 버튼을 클릭하세요.
'Repository name' 필드에 원하시는 저장소 이름을 영어로 입력하세요. (예: manus-backup-system)
저장소 종류를 **'Private'**으로 선택하여 다른 사람에게 코드가 보이지 않도록 설정하세요.
'Create repository' 버튼을 클릭하여 저장소 생성을 완료하세요.
2. Personal Access Token (PAT) 발급받기
GitHub 화면 우측 상단의 프로필 아이콘을 클릭한 후, 'Settings' 메뉴로 이동하세요.
왼쪽 메뉴에서 **'Developer settings'**를 클릭하세요.
'Personal access tokens' > **'Tokens (classic)'**을 선택하세요.
'Generate new token' > 'Generate new token (classic)' 버튼을 클릭하세요.
'Note' 필드에 토큰의 용도를 적으세요. (예: Manus Backup System)
'Expiration' (만료 기간)을 선택하세요. (보안을 위해 30일 또는 90일을 권장합니다.)
'Select scopes' (권한 범위)에서 repo 항목에 체크하세요. (저장소에 접근하고 코드를 업로드하는 데 필요합니다.)
페이지 하단의 'Generate token' 버튼을 클릭하세요.
생성된 토큰은 이 페이지를 벗어나면 다시 볼 수 없으니, 반드시 바로 복사하여 안전한 곳에 보관한 후 저에게 알려주세요. 토큰은 ghp_로 시작합니다.
위 안내에 따라 저장소 이름과 **Personal Access Token(PAT)**을 모두 준비하신 후 알려주시면, 백업 시스템 파일 업로드를 진행하겠습니다.
